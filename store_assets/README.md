# Store assets — Sanatan Guide v1.0.0

Checklist for first Play Store submission. Every artifact here ships to
either the Play Console listing or the in-store presentation.

## Status

| Asset | Path | Status |
|---|---|---|
| App title (30 chars) | `listing/title.txt` | ✅ draft |
| Short description (80 chars) | `listing/short_desc.txt` | ✅ draft |
| Full description (4000 chars) | `listing/long_desc.md` | ✅ draft |
| What's new (500 chars) | `listing/what_new.txt` | ✅ draft |
| Hi-res icon (512×512) | from adaptive icon | ✅ generated by build |
| Feature graphic (1024×500) | `feature_graphic/` | ⚙️ run prompt |
| Phone screenshots × 2-8 (1080×1920) | `screenshots/` | ⚙️ capture on device |
| Tablet screenshots (optional) | `screenshots/tablet/` | skip v1 |
| Privacy policy URL | hosted Gist | ✅ done |
| Terms URL | hosted Gist | ✅ done |
| Data safety form | Play Console only | ⚙️ Saurabh to fill |
| Content rating IARC | Play Console only | ⚙️ Saurabh to fill |
| App category | `Books & Reference` | decided |
| Pricing | Free, no IAP, no ads | decided |

## Workflow

1. Run `./scripts/release.sh <GEMINI_API_KEY>` → produces `app-release.aab`.
2. Capture screenshots per `screenshots/README.md`.
3. Generate feature graphic per `feature_graphic/prompt.md`.
4. Upload `.aab` to Play Console Internal Testing.
5. Paste listing copy from `listing/`.
6. Fill Data Safety + IARC questionnaires (one-time).
7. Soak 3 days on Internal Testing.
8. Promote to Production.

## Targets

- Android 7.0+ (API 24 = `flutter.minSdkVersion`).
- Target SDK = `flutter.targetSdkVersion` (currently 34).
- Languages: English (primary). Devanāgarī text inside, but UI is English.

## What we do NOT submit in v1

- iOS / App Store — Android-only first launch.
- Tablet-optimised screenshots — phone form factor only.
- Promo video — feature graphic + screenshots cover this.
- In-app review prompt — skip until 2 weeks of organic install data.
